✓ Done What's already working

✓
Maze generation — DFS (recursive backtracker)Mandatory
Implemented in mazegen/generator.py. Walls encoded as N=1, E=2, S=4, W=8. Full connectivity guaranteed, no isolated cells.
✓
Seed-based reproducibilityMandatory
SEED key supported in config. Same seed always produces the same maze. Passed through MazeGenerator.
✓
PERFECT flag — single path modeMandatory
When PERFECT=True, the maze has exactly one path between entry and exit. Extra wall-breaking is skipped.
✓
"42" logo embedded in mazeMandatory
Fully closed cells form the "42" pattern in the maze grid. Skipped with a console message if the maze is too small.
✓
No 3×3 open areasMandatory
Validated after generation. Corridors are max 2 cells wide. 2×3 and 3×2 areas are allowed, 3×3 is not.
✓
Hex output fileMandatory
Written row-by-row, one hex digit per cell. Followed by empty line, entry/exit coords, then BFS path as N/E/S/W letters.
✓
BFS solverMandatory
Shortest path from entry to exit computed in mazegen/solver.py. Result stored as expanded grid coordinates for rendering.
✓
Config file parserMandatory
Handles KEY=VALUE pairs, ignores # comments, validates all mandatory keys. Clear error messages on invalid config.
✓
MLX graphical displayMandatory
Full window rendering with tile images for walls, floors, trails, and the duck character. XPM asset pipeline (PNG → magick → XPM).
✓
Two visual themes — Light & DarkMandatory
Normal theme (light walls, wood floor, yellow duck) and Gothic theme (dark walls, stone floor, vampire duck). Selected at menu.
✓
Image-based menuMandatory
Background image with pre-drawn menu (menu_backgroundnew.png) displayed fullscreen. Key 1 = Light, Key 2 = Dark, ESC = Quit.
✓
MazeGenerator as reusable classMandatory
Implemented in mazegen/ as a standalone importable module. Accepts width, height, entry, exit, perfect flag, and optional seed.
✓
Makefile existsMandatory
Makefile is present. Needs verification that all required rules are implemented (see "Needs Check" section).

README.md existsReadme
README.md is present. Needs verification that all required sections are complete (see "Needs Check" section).

Poll mlx_mouse_get_pos in the loop hook. When the cursor is over a button area, swap to a slightly brighter version of the background image or draw a highlight overlay rectangle on top.

Multiple generation algorithmsBonus
The subject explicitly lists this as a bonus. Add Prim's or Kruskal's algorithm selectable via config key ALGORITHM=prim. Each produces mazes with very different visual character — Prim's creates more open areas, DFS creates long winding corridors.
★
Generation animationBonus
The subject explicitly lists this as a bonus. Instead of generating the full maze before displaying, step through the DFS frame by frame using the loop hook, rendering each wall removal as it happens. Visually stunning and shows algorithmic understanding.

Path animation — step by stepBonus idea
Instead of showing the full BFS path immediately, animate the trail tile appearing step by step along the path. Can be triggered by a key separate from the instant show/hide. Gives a satisfying "solve" moment.
